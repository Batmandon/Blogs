Inserting Data

Basic Insert Syntax

The standard SQL form for adding a row is INSERT INTO table (columns) VALUES (values).

Network Optimization (Round Trips)

Every interaction with the database incurs a network round-trip latency. Keep application and database close (same region/availability zone) when possible to reduce latency.

Bulk Inserts

Rather than issuing many single-row INSERT statements (each causing a round-trip), combine rows into one statement:

INSERT INTO table (col1, col2) VALUES
	('row1col1', 'row1col2'),
	('row2col1', 'row2col2'),
	('row3col1', 'row3col2');

This sends a single request and receives a single response, improving throughput.

Handling Default Values

PostgreSQL applies column defaults automatically (for example serial/identity IDs or timestamp defaults like created_at), so only provide the columns you need to set.

The RETURNING Clause

PostgreSQL supports returning data from the inserted row(s) without a separate SELECT call. Append RETURNING * to get all columns, or list specific columns:

INSERT INTO users (name, email)
VALUES ('Alice','alice@example.com')
RETURNING id, email;

This reduces round trips by combining insertion and retrieval into a single operation.
